Starter manual

Ship Hive dApps with a real foundation, not a demo scaffold.

Honeycomb gives you the stack that tends to matter in production: a clean Aurelia 2 shell, a typed API boundary, resumable indexing, and Hive auth flows that are already wired into the UI.

What this is for

New Hive products that need sensible defaults for auth, routing, `custom_json`, API shape, and indexing.

What to change first

Operation ids, payload contracts, route modules, and the starter pages. The base structure should stay.
